Polymorphism ->Static Polymorphism / Early binding / static binding - Compiler knows which one will execute (Overloading)
		
	     ->Dynamic Polymorphism / Late binding / dynamic binding  - the object will be changed at the running time 
		   - assigning more defined variable (rectangle) to a less defined variable (shape) is called covariance (Overriding)

  1. Function Overloading :
	A concept that i seen within a class
	A function with same name but different number of arguments or data types
	Signatures in terms of the arguments will differ

  2. Operator Overloading :
	To effectively be able to use operators that normally work with value types, also work with reference types

  3. Overridimg :
	is seen across related classes
	the function signatures have to be the same
	we cannot change access specifiers / modifiers

Overriding:

Virtual : It is used to provide the permission to the child class that the child can use its method


class SHape{
	protected float R, L, B;
	public virtual float Area(){ //virtual is used to provide permission to the child class
		return 3.14f * R * R;
	}
	
	public virtual float Circumference(){
		return 2 * 3.14f * R;
	}
}
