Readonly : ->variable/field that is declared using readonly keyword
	   ->we can initialize readonly variable/fields either at the time of declaration or through constructor
	   ->cannot be modified after initialization
	   ->constructor initialization has precedence
	   ->similar to non static members of the class (meaning a copy of the variable/field is maintained in each instance
	    ->they differ from non static as they cannot be modified unlike non-static field
	    ->it requires object to access it

Constants : -> same as readonly(once initialized cannot change)
	    -> unlike readonly we can initialize constants only at the time of declaration
	    -> are like static members, since there exists only one copy of the constant for the entire class
 	    -> while the values of static numbers can be changed, constants cannot be

Classes and objects :   -> Class is a structural declaration of what is needed and how to access it
			->Object is the working copy of the class(an instance of the class)
			     Data,State,behavior



Constructors :  ->are special functions of class
		->they have the same name as that of the class
		->they do not have return type
		->used to initialize fields
		->Constructor take precedence over initialization
		->They can be overloaded
		->they can be called explicitly like normal funtions using new keyword
		->they have to be either public, private, internal , protected but not private

Destructors :   ->special funation of the class
		->has the same name as that of that of the class with a tilde '~'
		->no return type
		->no access specifies
		->cannot be overloaded
		->cannot be called explicitly
		->are meant for deallocation of memory
