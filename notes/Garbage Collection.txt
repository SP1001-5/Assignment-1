Garbage Collection

	gc always works on managed heap
	it internally has an engine called optimization engine
	gc occurs if atleast one of the below condition is satisfied
	  1.if the system has low physical memory
	  2.if the memory allocates to various objects in the heap  exceeds a pre-set threshold
	  3.if forceful collection using GC.Collect() is issued/called

3 phases of GC:
  1.Marking Phase :
	List of all live objects that are created
  2.Re locating Phase : 
	Live objects refernce are updated
  3.Compacting Phase :
	Heap gets compacted and the dpace occupied by the dead objects are realesed

3 generations of GC:
  1. generation 0 :
	newly allocated objects
  2. generation 1 :
	objects that left after being in Generation 0  garbage
  3. generation 2 :
objects left after generation 1 garbage 
	
important methods of GC:
	1. GC.GetGeneration() : return the generation number
	2. GC.GetToatlMemory() : return the number of bytes allocated
	3. GC.Collect()

Dispose : 
	  it is usually written by our code
	  the resources are freed instantly
	  to implement dispose(), the class has to implement IDisposable inteface


Finalize  
	  is meant to be called by the GC and not by our

Using clause in C# :
	The usage of using has 3 functionalities
  1.namespace usage
  2.satic class usage
  3.code block usage : where the objects used are disposed automatically without explicit call to Dispose()
