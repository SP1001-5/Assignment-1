indexers 
	allow an object to be indexed like an array
	if we define an indexer for a class, then the class object behaves similar to a virtual array

	*Eg: cobj[0], cobj[2]
	indexers are like properties with get and set accessors
	unlike properties they do not have names and uses the 'this' keyword	
	while Properties get or set a specific datamember/field value, indexers get or set a value for a field at a particular index
	Can be overloaded
