Abstraction 

Abstract Class :
```````````````
	-> is a class that has atleast one abstract method
	-> can have non-abstract method also
	-> cannot be instantiated
	-> can be inherited
	-> the inherited class/s will have to provide implementation/definition/function body to the abstract method

Abstract method : 
`````````````````
	-> is a method that has only declaration/function signature and no implementation

Normal method:
``````````````
	-> it should be created with virtual key word

Mode of execution:
``````````````````` 

with Basetype refering the sub type then:
	- Area() of subtype will be executed
	- Circumference() of base type will be executed if not overrididden by the subtype, also subtypes overridden function will be executed
	- ShapeDetails() of base type will only be executed

with subtype object:
	- Area() of subtype will be executed
	- Circumference() of base type will be executed if not overrididden by the subtype, also subtypes overridden function will be executed
	- ShapeDetails() of sub type will only be executed


````````````````````````````````````````````````````````

abstract class Shapes{
	abstract public int Area();

	public virtual int CIrcumference(){
		return 0;
	}

	//non abstract non virtual method
	public void ShapeDetails(){
		Write("base");
	}
}

class Square : Shapes{
	int side = 0;
	public Square(int n){
		side = n;
	}

	//compulsory override
	public override int Area(){
		return side * side;
	}

	//optional override
	public override Circumference(){
		return 100;
	}
	public new void ShapeDetails(){
		Write("square");
	}
}
