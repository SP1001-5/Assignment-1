Exceptions
	->is an unwanted or unexpected event, which occurs during the execution of the program, ie at the runtime 
	->this disrupts the flow and exposes the code 
	->In order to avoid these events from exposing or disrupting the flow of execution, we have to handle them to come out very subtly

Handling Exception :
  try() - a block that identifies the code of probable exceptions
  catch() - handles the thrown exception
  finally() - is used to execute statements irrespective of exceptions or not
  throw - a jump statement meant to make sure of a relevant exception class and throw it to the source of the problem

Exception Classes in C#:
	->All exceptions are represented by classes
	->System.Exception is the base class for all exceptions
	->There are 2 derived types of exceptions
	 	1.System Exception - in built
		2.Application exception - user defined 

try is followed by one or more catch blocks only

User -> Exception Event -> an object of the exception class will be thrown to the program -> catch will handle the exception else the code disrupts

base exception (exception e) should be declared after every specific exception catches

finally should be declared after every exceptions

User defined Exceptions : 
  1. create Exception class
  2. decide when to raise an exception(code wise);
  3. Throw the exceptions
  4. handle the exceptions

		}
	}
	
}

User defined Exception:


class VotingExe : ApplicationException{
	public VotingExe(string msg : base(msg){
	}
}

class Vote{
	int age;
	public void AcceptAge(){
		Write("Age:");
		age = Convert.ToInt32(Console.ReadLine());
		if(age < 18){
			throw(new VotingExe("Age should be greater than 18"));
		}
		else{
			Write("Thanks for voting");
		}
	}	
}

class UserExceptions{
	static void Main(){
		Vote vote = new Vote();
		try{
			vote.AcceptAge();
		}
		catch (VotingExce e){
			Write(e.Message);
		}
		Console.Read()
	}
}
